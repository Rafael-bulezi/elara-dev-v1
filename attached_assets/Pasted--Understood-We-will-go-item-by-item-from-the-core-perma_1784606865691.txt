### Maximum Potential for User Accounts (What “Done Properly” Looks Like)

For a marketplace like Elara, User Accounts should feel **trustworthy, clear, and flexible**. Here’s what maximum potential includes:

#### A. Authentication (Login & Signup)
- Phone number + Password (primary, important in Angola)
- Google login (keep it)
- Optional: later add Apple or Facebook if needed
- Strong password rules + clear error messages
- “Forgot password” flow that actually works well
- Ability to login with phone number easily
- Session stays active properly (no random logouts)
- Guest browsing fully supported (can browse without account)

#### B. User Profile
- Clear profile page with:
  - Profile photo
  - Full name
  - Phone number (with verification status)
  - City / Location
  - Bio / Short description (especially useful for sellers & mixeiros)
  - Join date
  - Verification badges
- Ability to edit all important fields easily
- Avatar upload with compression and cropping

#### C. Roles System (Very Important)
- Every user starts as **Buyer**
- User can become **Seller** (with a clear “Become a Seller” flow)
- Later can also act as **Mixeiro**
- Clean way to switch between modes (Buyer Mode ↔ Seller Mode)
- Different menus and dashboards depending on the active role
- Role status clearly visible

#### D. Trust & Safety Elements in the Account
- Phone verification status (Verified / Not verified)
- Optional ID verification status (for sellers)
- Account creation date
- Number of successful transactions / reviews (later)
- Ability to report a user
- Block user option

#### E. Onboarding Experience
- Smooth first-time experience
- Collect city / province early (important for delivery and local feel)
- Simple explanation of how Elara works
- Encourage completing profile

#### F. Technical Quality
- Proper Supabase Auth implementation (secure, with correct session handling)
- Good loading and error states
- Mobile-first experience
- Fast and reliable
